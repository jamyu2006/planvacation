In published literature (and college essays), there will oftentimes come up the issue of ghost-writing -- hiring someone to write a piece for you and then putting your own name on it to take credit. Most notably, this has been studied with Shakespeare’s works, which were under scrutiny for being too large in quantity for one person to write alone. In order to test this, there’s a rather simple method used to calculate the similarity of two texts, relying on the frequencies of certain words used.

In this assignment, you will be creating a program which iterates through a folder of text files and uses this method to determine whether or not each text’s author is the same as the author of another text. The basic outline of the method is that the program will read each file, tally up the number of occurrences of each word (within a certain range of frequency, explained below) using a hash table, and then use the cosine method (see below) between each pair of hash tables to determine whether or not they have the same author. This will then be presented to the user at the end of the sequence.

Hashtable AND HashMap:

In this assignment, you may either implement your own hashing function or implement the provided Hashtable or HashMap classes for the FrequencyTable class. Both Hashtable and HashMap function very similarly: by mapping a value to a key using the put(key, value) method, and accessing that same value via the key using the get(key) method. Though they are almost identical for the purposes of this assignment, the most relevant difference is that while a HashMap may allow for one null key and as many null values as the user wants, a Hashtable will not allow for any null keys or values. Though the end result will be the same in this assignment, watch out for that discrepancy when testing your program. If you would like to know more about either of these classes, you should read the Oracle documentation for java.util.Hashtable and java.util.HashMap.

For the purposes of this assignment, there will never be negative values (because you can’t count a word into the negatives).

In the context of this assignment, u and v are the collections of frequencies of specific words in each text. For example, if a given word was found 40 times in a 1,000-word text, and 3 times in another 500-word text,  would be 40/1000 and  would be 3/500, repeated for each other word that was counted (note: occasionally there will be 0 occurrences of a given word in one text and >0 in another, and these would multiply to 0, which is intentional).

Typically, texts with the same author will be roughly 50-60% similar, sometimes more, whereas texts with different authors will typically be 30-40% similar, or lower.

2. FrequencyList

The frequencies of each given word in each text will be stored in an extension of ArrayList<Integer>. When constructed, this class will search through the passed ArrayList of Passages and find the occurrences of its given word in each Passage object, constructing an Integer list as well as a Hashtable/Map which will map Passage titles to their index in said Integer list- if (“Frankenstein”, 2) appears in the table, then FrequencyList.get(2) will return whatever value corresponds to “Frankenstein”. This class will contain a String member variable corresponding to its word that it’s keeping track of, and because some texts will have words that other texts don’t contain, this class will only hold the values which are greater than 0 to avoid taking up unnecessary space.

Note: You can store the frequency as doubles instead of the number of occurrences as integers (i.e. if there are 1,000 words in a text and 30 occurrences of a given word, you can store either 30 or 0.03; you will need the frequency regardless).
